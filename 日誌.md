### Phase 1 — 讀取參考腳本（完成）

**目標：** 從現有 Linux/Python 腳本提取 HID 協議細節，避免從零開始 reverse engineering

**Clone 的參考 repo（存放於 `reference/`）：**
- `ikz87/NSW2-controller-enabler` → `enable_hid.py`
- `djedditt/nsw2usb-con` → `nsw2usb-con.py`

**從腳本提取的關鍵資訊：**

#### Product IDs
- NS2 Pro Con：VID `0x057E`，PID `0x2069`（djedditt 確認）
- NS2 GC NSO：VID `0x057E`，PID `0x2073`（ikz87 + djedditt）
- USB Interface：`1`

#### Init 指令序列（PID 0x2069 最小版本，djedditt 確認）
```
Step 1 INIT:         03 91 00 0D 00 08 00 00 01 00 FF FF FF FF FF FF
Step 2 Player 1 LED: 09 91 00 07 00 08 00 00 01 00 00 00 00 00 00 00
延遲：每個指令間 50ms
```
- ikz87 的完整版有 17 個指令（針對 GC NSO variant），前兩個與 djedditt 完全相同
- 策略：先用 2 個指令版本測試；若 LED 不亮再加入完整序列

#### Input Report 格式（64 bytes）
```
report[0]    = Report ID（跳過）
report[1..2] = unknown
report[3]    = Button byte 0：B/A/Y/X/R/ZR/Plus/RStick
report[4]    = Button byte 1：DPad4方向/L/ZL/Minus/LStick
report[5]    = Button byte 2：Home/Capture
report[6..8] = Left Stick（12-bit packed，3 bytes → 2 x 12-bit）
report[9..11]= Right Stick（同上）
report[13]   = Left Trigger（raw 36-240 → remap 0-255）
report[14]   = Right Trigger（同上）
```

#### Stick 解碼（12-bit packed）
```cpp
a = bytes[0] | ((bytes[1] & 0x0F) << 8);  // X 軸
b = (bytes[1] >> 4) | (bytes[2] << 4);    // Y 軸
// 中心值 = 2048，範圍 0-4095
// 轉 8-bit：value >> 4
```

#### Trigger 重映射
```
原始範圍：36–240
輸出範圍：0–255
公式：((raw - 36) / 204.0) * 255
```

**輸出：** `reference/protocol_notes.md` — 作為 C++ 實作的單一真實來源

**Rumble 狀態：** 兩個腳本都沒有實作 rumble。Phase 4 另行處理。

### Phase 2 — 建構環境（完成）

**目標：** 加入 ViGEmClient，讓專案可以成功編譯

**ViGEmClient 取得方式：**
- NuGet 版本只含 .NET binding，不含 C++ headers → 捨棄
- 改用 `git clone https://github.com/nefarius/ViGEmClient.git third_party/ViGEmClient`
- ViGEmClient 有自己的 `CMakeLists.txt`，用 `add_subdirectory` 直接整合

**CMakeLists.txt 更新：**
```cmake
add_subdirectory(third_party/ViGEmClient)
target_link_libraries(ns2drv PRIVATE
    hidapi::hidapi
    ViGEmClient::ViGEmClient
    setupapi
)
```

**Build 指令：**
```powershell
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake -DCMAKE_BUILD_TYPE=Debug
cmake --build build --config Debug
```

**結果：**
- CMake configure：成功（MSVC 19.44，Windows SDK 10.0.26100）
- Build：成功，輸出 `build/Debug/ns2drv.exe`
- Warning：`InputParser.cpp` 含中文注解，codepage 950 警告（不影響功能）

**問題診斷：**
- 第一次執行：`HID device opened successfully` 但 `Failed to send INIT command`
- 根本原因：`hid_write()` 只能存取 Interface 0（HID），但 init 指令必須送到 **Interface 1（bulk endpoint）**
  - 參考腳本都用 pyusb (libusb) 送 init，再用 hidapi 讀 input
  - 我們最初只用 hidapi，路徑錯誤

**架構修正（對應 Python 腳本的兩階段做法）：**
```
Step 1: libusb → Interface 1 → bulk OUT → 送 init 指令 → 釋放
Step 2: hidapi → Interface 0 → HID read → 讀 input reports
```

**新增檔案：**
- `src/hid/UsbDevice.hpp/cpp` — libusb wrapper，負責 Interface 1 bulk write
- `HidInitializer` 改用 `UsbDevice` 而非 `HidDevice`
- `NS2Controller::connect()` 加入兩階段流程 + 2 秒等待（同 ikz87 腳本）

**libusb 加入：**
- `vcpkg install libusb:x64-windows` → 1.0.29
- CMakeLists.txt 用 `find_path` + `find_library` 直接找 vcpkg installed 路徑
  （`pkg_check_modules IMPORTED_TARGET` 在 MSVC+vcpkg 下 include path 未正確傳遞，改用手動方式）

**Build 結果：** 成功，`build/Debug/ns2drv.exe`

**待完成：**
- [ ] 安裝 Zadig，為 Interface 1 裝 WinUSB driver（libusb 在 Windows 存取 Interface 1 的必要步驟）
- [ ] 接上 controller 測試 init 序列（LED 亮起）

### Phase 3 — ViGEmClient 虛擬手把（完成）

**按鈕映射（NS2 → Xbox 360，位置對應）：**
- A(右) → B，B(下) → A，X(上) → Y，Y(左) → X
- L/R → LB/RB，ZL/ZR → LT/RT（數位轉255）
- Plus → Start，Minus → Back，Home → Guide
- L-stick/R-stick click → LS/RS
- DPad 四方向 → DPad
- 背鍵左右 → 未映射（Xbox 360 無對應）

**搖桿軸確認（從原始資料）：**
- 推上 → LY raw 增加（UP=bigger），符合 XInput 正值=上的慣例
- 不需要反轉 Y 軸

**熱插拔：**
- 外層 loop 每 500ms 用 `hid_enumerate` 偵測裝置
- 插入 → init → 建立虛擬手把 → 開始讀取
- 拔出 → 移除虛擬手把 → 回到等待

**Ctrl+C 乾淨關閉：**
- `SetConsoleCtrlHandler` 捕捉 CTRL_C/BREAK/CLOSE
- 呼叫 `controller.stop()` → 正常退出 loop → ViGEm 正確移除

**HidHide 設定：**
- 手動設定（Devices 勾選 NS2 HID interface，Applications 加入 ns2drv.exe）
- 採用「永久隱藏」策略：exe 不在時 if_hid 依然隱藏（使用者接受此行為）
- 嘗試程式化控制 IOCTL_SET_ACTIVE 但 IOCTL 失敗，維持手動設定

**延遲優化：**
- 原本：非阻塞 read + 100ms timeout + 1ms sleep → 最差 101ms
- 修正：blocking read（`hid_set_nonblocking=0`），timeout 100ms 僅用於檢查 running_ flag
- 結果：延遲降至接近手把原生 8ms report rate

### Phase 4 — 打包與系統托盤（完成）

**目標：** 讓普通使用者可以一鍵安裝，並在背景執行

#### 4-1 WIN32 GUI 模式（無 console 視窗）

- `CMakeLists.txt` 改為 `add_executable(ns2drv WIN32 ...)`，入口點改為 `WinMain`
- 建立 message-only window（`HWND_MESSAGE`），使用者看不到任何視窗
- NS2Controller 在背景 thread 執行，主執行緒跑 Windows 訊息迴圈

#### 4-2 系統托盤圖示

**新增檔案：**
- `src/ui/TrayIcon.hpp/cpp` — 封裝 `NOTIFYICONDATAW`、右鍵選單、tooltip

**圖示來源：**
- 使用者提供 `NS2Driver.png`（手把圖案）
- 用 Python Pillow 轉成 `NS2Driver.ico`（含 16/24/32/48/64/128/256 px 多尺寸）
- 嵌入 exe：`src/ns2drv.rc`（`1 ICON "..\\NS2Driver.ico"`）
- TrayIcon 以 `LoadIconW(GetModuleHandleW(nullptr), MAKEINTRESOURCEW(1))` 載入

**右鍵選單功能：**
- `Launch at startup` — 可勾選，切換開機自動啟動
- `Exit NS2 Driver` — 結束程式

**controller 狀態 → tooltip 回報：**
- `onStatusChange_` callback 在每次狀態變更時更新 tooltip 文字

#### 4-3 開機自動啟動

- 使用 Task Scheduler（`schtasks`），而非登錄檔 Run key
- 理由：Run key 每次啟動需要 UAC 確認，Task Scheduler 可以靜默以最高權限啟動
- 任務參數：`/SC ONLOGON /RL HIGHEST /IT`

**從托盤切換啟動設定：**
- 查詢是否啟用：讀取 `C:\Windows\System32\Tasks\NS2DriverAutoStart` 是否存在（不需要提升，即時，不跳 UAC）
- 變更設定：用 `ShellExecuteExW` 執行 `schtasks /Create` 或 `/Delete`；若程式已以管理員執行則不跳 UAC，否則彈一次 UAC

#### 4-4 Inno Setup 安裝包

**打包工具：** Inno Setup 6（安裝於 `%LOCALAPPDATA%\Programs\Inno Setup 6\`）

**安裝包內容（`installer/dist/NS2DriverSetup.exe`）：**
| 元件 | 說明 |
|------|------|
| `ns2drv.exe` | 主程式 |
| `hidapi.dll` | HID 通訊 |
| `libusb-1.0.dll` | USB bulk write |
| `ViGEmBusSetup.exe` | 虛擬手把驅動（若未安裝才執行） |
| `HidHideSetup.exe` | 裝置隱藏驅動（若未安裝才執行） |
| `configure_hidhide.ps1` | 自動設定 HidHide 允許清單與隱藏清單 |

**安裝流程（全自動）：**
1. 安裝 ViGEmBus（`/quiet /norestart`，已裝則跳過）
2. 安裝 HidHide（`/quiet /norestart`，已裝則跳過）
3. 執行 `configure_hidhide.ps1`：加入 ns2drv.exe 白名單、隱藏 NS2 HID 裝置、啟用遮蔽
4. 建立 Task Scheduler 開機啟動任務
5. 立即啟動程式
6. 若 HidHide 為新安裝，最後提示使用者重開機

**解除安裝：**
- 停止程式（`taskkill`）
- 刪除 Task Scheduler 任務
- 詢問是否一同解除安裝 ViGEmBus、HidHide（各自獨立 Yes/No 對話框）

**打包腳本：** `installer/prepare.bat`
- 自動從 `build/Release/` 複製最新 exe + DLL
- 呼叫 `ISCC.exe` 編譯 `ns2drv_setup.iss`

#### 4-5 已修正的 Bug

| Bug | 原因 | 修正 |
|-----|------|------|
| 托盤 icon 空白 | RC 用名稱 `IDI_APPICON`，程式用數字 `1` 載入，不符 | RC 改為 `1 ICON ...` |
| HidHide 安裝後靜默重開機 | `/quiet` 未抑制重啟，導致後續步驟（Task Scheduler 設定）未執行 | 改為 `/quiet /norestart`，Inno Setup 最後統一提示 |
| 開機啟動任務未建立 | HidHide 重開機發生在 Task Scheduler 建立之前 | 同上 |
| 已插著的手把 init 失敗 | 開機時 Windows 已佔用 USB Interface | 加入 `libusb_reset_device()` + 1.5s 等待 + retry |
| 右鍵每次跳 UAC | `isAutostartEnabled()` 呼叫 `schtasks /Query`（走 runas） | 改為直接讀 `System32\Tasks\` 檔案存在與否 |
